# License Audit Flowchart — Reference

Loaded by `linguistic-ethics` Step 2 + Step 3 (mix audit).

## Per-dataset license decision

```
1. Is there a LICENSE file or explicit license statement?
   ├── NO → treat as restricted; contact source; default deny.
   └── YES → continue.

2. Identify the license:
   ├── Public Domain / CC0       → no attribution required (but check CARE).
   ├── CC-BY                     → attribution required; track in registry.
   ├── CC-BY-SA                  → attribution + ShareAlike PROPAGATES to model output.
   ├── CC-BY-NC                  → no commercial use; check entire training mix.
   ├── CC-BY-NC-SA               → both restrictions.
   ├── CC-BY-ND                  → no derivatives; usually BLOCKS training (training is derivation).
   ├── ODbL                      → database license; SA-like for derived data.
   ├── ODC-BY                    → attribution-required for the dataset.
   ├── MIT / Apache 2 / BSD      → permissive; usually for code/tools, not data.
   ├── GPL / AGPL                → copyleft for code; rarely for datasets.
   ├── Custom community (Māori Data License, Kaitiakitanga, etc.) → respect terms; community-gated.
   ├── Academic-use-only         → no commercial training.
   ├── "All rights reserved"     → don't use.
   └── Unknown / ambiguous       → contact source; default deny.

3. For CC-BY family: capture attribution string in attribution_registry.md.

4. For NC: check whether your downstream use is commercial.
   - "Free chatbot funded by ad revenue" — commercial.
   - "Internal employee tool" — usually commercial (depends on jurisdiction).
   - "Academic research only" — non-commercial OK.

5. For SA: track that propagation obligation reaches model output / downstream.
```

## Mix-license matrix

When training mixes N datasets, the model inherits the most-restrictive terms.

| Has any CC-BY-NC? | YES → entire model is NC. |
| Has any CC-BY-SA? | YES → model output must propagate SA. |
| Has any CC-BY-ND? | YES → you've already violated ND by training; STOP. |
| Has any custom community license? | YES → likely community-gated release, check per-license. |
| Has any unknown / restricted? | YES → STOP. |

## Common mix-licensing gotchas

1. **The "open" Pile mix**: includes some CC-BY-NC sub-corpora. Many widely-used Pile derivatives are technically commercial-use-violating.
2. **CulturaX / MADLAD-400 / Glot500**: aggregate from many sources. The aggregator may be open-licensed but per-source licenses must be checked. (NLLB published these distinctions; not all derivatives propagate.)
3. **Common Crawl segments**: CC's TOS is permissive but per-page rights of crawled content are NOT. Newspapers in particular routinely sue.
4. **Academic datasets that became commercial-defaults**: SQuAD, GLUE, etc. — many have non-commercial restrictions for their constituent sources.
5. **Bible-NLP**: open-licensed translations widely used, but many communities consider commercial generative AI use to violate norms even when license permits. CARE > license here.
6. **GitHub code datasets**: per-file licenses in the repo apply; "MIT-licensed dataset of GitHub" is ambiguous (the dataset IS open; the file licenses are heterogeneous).

## License → Model output license matrix (output-side)

After training, what license can the model output / weights carry?

| Training mix has | Model release options |
|---|---|
| All CC0 / Public Domain | Any license (including proprietary) |
| All permissive (CC-BY family without NC/SA/ND) | Permissive; keep per-source attribution |
| Any CC-BY-SA | Propagate SA to model output license |
| Any CC-BY-NC | Non-commercial only |
| Any CC-BY-NC-SA | Non-commercial AND ShareAlike |
| Any CC-BY-ND | Don't release — you've already violated ND by training |
| Any custom community license | Per-license — usually community-gated |

## Attribution requirements

For CC-BY family + per-source attribution:

```markdown
## Model Card — Attribution

This model was trained on data including:
- [Dataset 1] (CC-BY 4.0) — Author/Org, year. Source: <URL>
- [Dataset 2] (CC-BY-SA 4.0, propagated) — Author/Org, year. Source: <URL>
...
```

For Indigenous / community datasets, follow community attribution conventions which may be richer than license requires (e.g., naming the elders, naming the community, citing the documenter and the speakers separately).

## Special cases

- **Synthetic data generated by another model**: attribution + license depend on the generator's license (e.g., GPT-4 outputs have OpenAI ToS that affect downstream).
- **User-contributed data via app**: privacy + consent + per-user terms apply on top of any service license.
- **Crowdworker-collected data** (Mechanical Turk, Prolific): worker rights + data processor obligations apply; the platform license isn't sufficient.
- **Government / public records**: usually open but jurisdiction-specific (US public records ≠ EU public records).

## Refresh procedure

License landscape changes; update this reference annually:
- Creative Commons license version updates (current: 4.0).
- New community licenses (Māori Data License revisions; new Indigenous data licenses).
- Court rulings affecting training-data law (US Authors Guild v. OpenAI; EU AI Act decisions).
- Major aggregator license clarifications (NLLB, Common Crawl).
